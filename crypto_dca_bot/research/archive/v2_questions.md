## 1. Pionex 罐頭機器人能不能做你想的策略?

> 截圖 source:Pionex App → 創建機器人(2026-05-09 觀察)

### Pionex 能做的(現有機器人類型)

**網格類(Grid 自動低買高賣)**
- 現貨網格、合約網格、全倉合約網格、幣本位合約網格、鏈上網格
- 進階:借貸網格、槓桿網格、無限網格、反向網格

**定投 / 屯幣**
- 定投機器人(DCA,V1 等同這個)
- 屯幣寶(累積型 DCA)

**馬丁格爾 / 加碼類(Martingale 虧損加倍下注)**
- 馬丁格爾、合約馬丁、追蹤馬丁

**追蹤類(Trailing 移動止盈/止損)**
- 追蹤買入、追蹤賣出、波段追蹤

**機會 / 觸發類**
- 抄底寶、止盈寶、幣幣抄底止盈、震盪寶
- 抄底止盈工具

**限價單(Limit Order)**
- 限價止盈止損、比特幣天地單、以太幣天地單、EB 天地單、合約天地單

**避險 / 套利**
- 套保機器人

**訊號 / 跟單**
- 訊號機器人(吃外部 signal 訊號執行)
- 帶單 / 跟單機器人(複製他人交易)

### Pionex 不能做的(= V2 存在空間)

1. **自訂訊號生成**:訊號機器人**吃外部 signal,但 Pionex 不會幫你產 signal**。要做基於自家邏輯的策略,還是得自己寫產生訊號的引擎(engine 引擎)。
2. **鏈上資料驅動決策**:「鏈上網格」只是在 DEX(去中心化交易所)上做網格,不是基於鏈上指標如 MVRV(市場價值/已實現價值比)、whale transfer(巨鯨轉帳)、SOPR(花費輸出獲利率)做進出場。
3. **多幣輪動 / 相對強度策略**:Pionex 是單幣機器人,不會做「BTC vs ETH 強弱切換」這種跨幣判斷。
4. **新聞 / 情緒驅動**:沒有 LLM 讀新聞、社群情緒、宏觀事件做決策的功能。
5. **多指標組合**:不能組合「RSI + 量能 + on-chain 指標 + 宏觀」當進場條件。每個罐頭是固定邏輯。
6. **Regime detection(市場狀態偵測)**:無法根據「趨勢市 / 盤整市 / 崩盤期」自動切換策略邏輯。
7. **跨交易所資料**:套保機器人只在 Pionex 內部跨產品避險,不會從 Binance / Coinbase 等拉資料做判斷。
8. **自訂回測驗證過的策略邏輯**:不能上傳自己寫的 code,只能填罐頭提供的參數。

### 結論方向

Pionex **執行層罐頭夠多**(各種交易動作都有),**但「決策層」不開放**。

V2 的存在空間 = **自己做決策層**:輸入資料 → 算 signal → 給機器人執行(或直接用 Binance API 出手)。

V2 候選策略應該圍繞:
- **資料優勢**:鏈上 / 新聞 / 跨市場相關性等 Pionex 沒接的資料源
- **判斷優勢**:多指標組合 / regime detection / 多幣輪動等 Pionex 罐頭做不到的邏輯
